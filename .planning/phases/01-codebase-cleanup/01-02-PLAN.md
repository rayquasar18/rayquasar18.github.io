  <name>Task 1: Rewrite LoadingScreen with proper timing and scale+fade exit</name>
  <files>src/components/LoadingScreen.tsx</files>
  <action>
    Rewrite LoadingScreen to fix all timing bugs per user decision. The component must:

    **Dismiss trigger (per locked decision):**
    - Wait for BOTH: `document.readyState === 'complete'` AND minimum 1.5s display time (whichever is later)
    - Use `Promise.all([readyPromise, minTimePromise])` pattern -- NOT separate state variables racing each other
    - Add a timeout fallback (8 seconds) using `Promise.race` so the loading screen never gets permanently stuck
    - Handle the case where readyState is already 'complete' when useEffect runs (dev mode Fast Refresh)

    **Exit animation (per locked decision):**
    - Use framer-motion `motion.div` with scale-down + fade-out (NOT the current h-screen to h-0 height transition)
    - Animate `opacity: 1 -> 0` and `scale: 1 -> 0.95` with `transition: { duration: 0.6, ease: "easeInOut" }`
    - Use the `onAnimationComplete` callback to detect when exit animation finishes
    - After exit animation completes, return `null` (remove from DOM entirely)

    **Scroll lock (Claude's discretion -- implementing recommended approach):**
    - Set `document.body.style.overflow = "hidden"` on mount
    - Restore `document.body.style.overflow = ""` when exiting or on cleanup

    **Component state machine:**
    - Three phases: `"loading"` -> `"exiting"` -> `"done"`
    - `"loading"`: fixed overlay visible with spinners
    - `"exiting"`: scale+fade animation playing
    - `"done"`: return null (removed from DOM)

    **Keep unchanged (per locked decision):**
    - ldrs spinners: Bouncy (size=45, speed=1.75) and Reuleaux (size=37, stroke=5)
    - Their CSS imports
    - The `"use client"` directive
    - White background, centered layout, z-50

    **Performance note (per fixing-motion-performance skill):**
    - Use transform and opacity for exit animation (compositor-only properties)
    - Do NOT animate height, width, padding, or any layout-triggering property

    Reference the code example in 01-RESEARCH.md "Example 1: LoadingScreen Rewrite Pattern" for the exact implementation pattern.
