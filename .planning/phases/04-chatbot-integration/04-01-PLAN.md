**src/types/chat.ts** -- Create the type contracts for the entire chat system:

1. `ChatMessage` interface:
   - `id: string` (generated via `crypto.randomUUID()`)
   - `role: 'user' | 'assistant' | 'system'` (system for greeting/error messages)
   - `content: string`
   - `emotion?: RobotEmotion` (import from `@/types/robot` -- only set on assistant messages)
   - `timestamp: number` (Date.now())

2. `SSEEvent` discriminated union type:
   - `{ type: 'emotion'; emotion: RobotEmotion }` -- first event in stream
   - `{ type: 'text'; content: string }` -- progressive text chunks
   - `{ type: 'done' }` -- stream complete signal (optional, `[DONE]` string also handled)

3. `ChatState` interface (store shape):
   - `messages: ChatMessage[]`
   - `isOpen: boolean`
   - `isStreaming: boolean`
   - `error: string | null`
   - Action methods: `addMessage`, `appendToLastMessage`, `clearMessages`, `setOpen`, `setStreaming`, `setError`, `resetToGreeting`

4. `ChatConfig` -- exported constants:
   - `MAX_MESSAGES = 50`
   - `RATE_LIMIT_COOLDOWN_MS = 2000` (2 seconds cooldown after send)
   - `STREAM_TIMEOUT_MS = 30000` (30 seconds)

**src/stores/useChatStore.ts** -- Create the Zustand store with persist middleware:

1. Use `create` from `zustand` with `persist` and `createJSONStorage` from `zustand/middleware`
2. No `'use client'` directive (same pattern as useRobotStore -- pure module, not a hook at definition time)
3. Initial state: `messages: []`, `isOpen: false`, `isStreaming: false`, `error: null`
4. Actions:
   - `addMessage(msg: Omit<ChatMessage, 'id' | 'timestamp'>)`: Adds message with auto-generated id and timestamp. After adding, prune to keep only the last `MAX_MESSAGES` entries (FIFO).
   - `appendToLastMessage(chunk: string)`: Finds the last message with `role === 'assistant'`, appends `chunk` to its `content`. If no assistant message exists at end, this is a no-op.
   - `clearMessages()`: Sets `messages: []`.
   - `setOpen(open: boolean)`: Sets `isOpen`.
   - `setStreaming(streaming: boolean)`: Sets `isStreaming`.
   - `setError(error: string | null)`: Sets `error`.
   - `resetToGreeting()`: Clears messages and sets error to null. (The greeting message itself is NOT stored -- it is rendered conditionally by the UI when messages array is empty.)
5. Persist config:
   - `name: 'rayquasar-chat'` (localStorage key)
   - `storage: createJSONStorage(() => localStorage)`
   - `partialize`: Persist ONLY `messages`. Do NOT persist `isOpen`, `isStreaming`, `error` -- these are transient UI state that must reset on page load.

**src/services/chat.ts** -- Create the chat service module (NO `'use client'` -- this is a pure TS module):

1. **Environment config** at module top:
   ```
   const LLM_API_URL = process.env.NEXT_PUBLIC_LLM_API_URL ?? '';
   const LLM_API_KEY = process.env.NEXT_PUBLIC_LLM_API_KEY ?? '';
   export const isApiConfigured = Boolean(LLM_API_URL && LLM_API_KEY);
   ```

2. **`ChatError` class** extending Error with `status?: number` field for HTTP error codes.

3. **`streamSSE` async generator** -- the core SSE parser:
   - Signature: `async function* streamSSE(url: string, body: { message: string }, apiKey: string, signal: AbortSignal): AsyncGenerator<SSEEvent>`
   - Sends `fetch(url, { method: 'POST', headers: { 'Content-Type': 'application/json', 'Authorization': 'Bearer ${apiKey}', 'Accept': 'text/event-stream' }, body: JSON.stringify(body), signal })`
   - If `!response.ok`, throw `ChatError` with status code
   - Uses `response.body!.getReader()` + `TextDecoder` to read chunks
   - Buffers partial lines: split on `\n`, keep last element as buffer (may be incomplete)
   - For each complete line starting with `data: `: if data is `[DONE]`, return. Otherwise `JSON.parse(data)` and yield as `SSEEvent`.
   - Wrap reader in `try/finally` that calls `reader.releaseLock()`

4. **`mockStreamChat` async generator**:
   - Signature: `async function* mockStreamChat(): AsyncGenerator<SSEEvent>`
   - Array of 5+ mock responses, each with `{ text: string, emotion: RobotEmotion }`. Vary emotions across happy, excited, sad, thinking. Include responses about Quan being an AI Engineer, his projects, his tech stack, and fun personality lines for the robot.
   - First yield: `{ type: 'emotion', emotion: response.emotion }`
   - Then split text into words, yield each as `{ type: 'text', content: word + ' ' }` with 50-100ms random delay between words (simulate streaming feel)

5. **`sendMessage` orchestrator function** -- THE main function called by UI:
   - Signature: `async function sendMessage(message: string): Promise<void>`
   - This function orchestrates the full send flow using store `getState()` calls:
     a. Add user message to chat store: `useChatStore.getState().addMessage({ role: 'user', content: message })`
     b. Set streaming state: `useChatStore.getState().setStreaming(true)` and `useChatStore.getState().setError(null)`
     c. Set robot to thinking: `useRobotStore.getState().setEmotion('thinking')`
     d. Create placeholder assistant message: `useChatStore.getState().addMessage({ role: 'assistant', content: '' })`
     e. Determine stream source: if `isApiConfigured`, use `streamSSE` with retry. If not, use `mockStreamChat`.
     f. Create `AbortController`. Combine with `AbortSignal.timeout(STREAM_TIMEOUT_MS)` via `AbortSignal.any([controller.signal, AbortSignal.timeout(STREAM_TIMEOUT_MS)])`.
     g. Try streaming. For each yielded event:
        - If `type === 'emotion'`: call `useRobotStore.getState().setEmotion(event.emotion)` and set emotion on the assistant message
        - If `type === 'text'`: call `useChatStore.getState().appendToLastMessage(event.content)`
     h. **Auto-retry logic**: Wrap the stream consumption in try/catch. On first failure (any error), retry once by calling the same stream source again with a fresh AbortController+timeout. If retry also fails, fall through to error handling.
     i. **Error handling**: On final failure, set robot emotion to `'sad'`, add a system error message to chat store via `addMessage({ role: 'system', content: errorMessage })`. Remove the empty placeholder assistant message if content is still empty. Set `error` in store.
     j. **Finally block**: Always set `useChatStore.getState().setStreaming(false)`.
   - Export `sendMessage` and also export `abortCurrentStream` function that calls `controller.abort()` on the current AbortController (stored in module-level variable). This allows cancelling on panel close or new message.
