Create the `src/components/chat/` directory and all 6 components. ALL components need `'use client'` directive.

**ChatBar.tsx** -- The always-visible entry point + panel container:
- Fixed position at bottom of viewport: `fixed bottom-0 right-0 z-50`
- Contains TWO states:
  - **Collapsed**: A slim bar at bottom-right. On desktop: `w-[400px]` anchored bottom-right with `right-4 bottom-4`. On mobile: `w-full bottom-0`. Shows a text placeholder ("Type a message..." from i18n) and a send icon (MessageCircle from lucide-react). Clicking expands the panel.
  - **Expanded**: The ChatPanel component slides up above the bar.
- Uses `useChatStore` to read/write `isOpen` state.
- On expand: `useChatStore.getState().setOpen(true)`. On collapse: `useChatStore.getState().setOpen(false)` and call `abortCurrentStream()` to cancel any active stream.
- Add `pb-[env(safe-area-inset-bottom)]` for iOS safe area on mobile.
- Import and render `ChatPanel` conditionally (via AnimatePresence from framer-motion for mount/unmount animation).

**ChatPanel.tsx** -- The expanded overlay chat window:
- Animated with framer-motion: `initial={{ y: '100%', opacity: 0 }}`, `animate={{ y: 0, opacity: 1 }}`, `exit={{ y: '100%', opacity: 0 }}`, `transition={{ type: 'spring', damping: 25, stiffness: 300 }}`.
- Layout:
  - **Header**: Title ("Robot Assistant" or similar -- use i18n), close button (X icon from lucide-react), clear button (Trash2 icon). Clear calls `useChatStore.getState().resetToGreeting()`.
  - **Message list**: Scrollable container (`overflow-y-auto`) with max height (`max-h-[60vh]` on desktop, `max-h-[70vh]` on mobile). Maps over `messages` from `useChatStore`. Auto-scrolls to bottom on new messages using `useRef` + `scrollIntoView({ behavior: 'smooth' })` triggered by `useEffect` watching `messages.length` and last message content length.
  - **Greeting state**: When `messages.length === 0`, show the greeting message (from i18n `Chat.greeting`) as a system-style bubble, followed by `PromptChips`.
  - **Typing indicator**: When `isStreaming` is true, render `TypingIndicator` at the bottom of the message list.
  - **Demo notice**: When `!isApiConfigured`, show a subtle banner at top: "Demo mode -- responses are pre-written" (from i18n `Chat.demoNotice`) with muted styling.
  - **Footer**: The `ChatInput` component.
- Styling: `bg-surface-elevated border border-border rounded-t-xl` (rounded top corners). Width matches ChatBar (400px desktop, full mobile).

**ChatBubble.tsx** -- Individual message rendering:
- Props: `message: ChatMessage`
- Three visual styles based on `message.role`:
  - `user`: Right-aligned, `bg-accent text-surface-base` (green bubble, dark text), `rounded-2xl rounded-br-sm`
  - `assistant`: Left-aligned, `bg-surface-overlay text-text-primary`, `rounded-2xl rounded-bl-sm`
  - `system`: Center-aligned, `text-text-muted text-sm italic`, no bubble background
- Content renders as plain text (no markdown parsing needed for v1).
- Max width: `max-w-[80%]` to prevent full-width bubbles.
- Subtle padding: `px-4 py-2`.

**ChatInput.tsx** -- Text input + send button:
- Controlled input with local `useState` for the text value.
- `form` wrapper with `onSubmit` handler that:
  a. Prevents default
  b. Trims input -- if empty, return
  c. Calls `sendMessage(input)` from `@/services/chat`
  d. Clears input
  e. Starts rate limit cooldown: set `isRateLimited` state to true, `setTimeout` to clear it after `RATE_LIMIT_COOLDOWN_MS`
- Send button: Uses `Send` icon from lucide-react. Ghost button style (border-accent, hover:bg-accent hover:text-surface-base).
- Disabled states: Disable send button when `isStreaming || isRateLimited || input.trim() === ''`. Show visual disabled state (opacity-50, cursor-not-allowed).
- Input styling: `bg-surface-overlay border border-border rounded-lg` with focus ring `focus:border-accent focus:ring-1 focus:ring-accent`.
- On Enter key: submit (default form behavior). No Shift+Enter handling needed (single-line input).
- Placeholder text from i18n: `Chat.placeholder`.

**TypingIndicator.tsx** -- Bouncing dots animation:
- Three dots that bounce in sequence with staggered animation.
- Use framer-motion for the bounce: `animate={{ y: [0, -6, 0] }}` with `transition={{ duration: 0.6, repeat: Infinity, repeatDelay: 0.1 }}` and staggered `delay` for each dot (0, 0.15, 0.3).
- Dots: 8px circles, `bg-text-muted rounded-full`.
- Wrapped in a left-aligned bubble container (same style as assistant bubble) to look like "assistant is typing".

**PromptChips.tsx** -- Suggested prompts for empty state:
- Props: `onSelect: (prompt: string) => void`
- Renders 3 chips using i18n keys `Chat.promptChip1`, `Chat.promptChip2`, `Chat.promptChip3`.
- Each chip: ghost button style matching existing codebase pattern (`rounded-md border border-accent px-3 py-1.5 text-sm text-accent hover:bg-accent hover:text-surface-base transition-colors`).
- Clicking a chip calls `onSelect(chipText)` which triggers `sendMessage(chipText)`.
- Horizontal flex wrap layout with gap-2.

**Styling notes for all components:**
- Use ONLY existing design tokens from globals.css (surface-base, surface-elevated, surface-overlay, text-primary, text-secondary, text-muted, accent, border, etc.)
- Use Tailwind utility classes -- no custom CSS files
- Font: inherit from body (font-body/MarlinGeoSQ) -- no need to specify
- All text via `useTranslations('Chat')` from next-intl
