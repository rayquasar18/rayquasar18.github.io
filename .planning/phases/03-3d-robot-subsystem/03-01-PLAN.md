  <files>src/types/robot.ts, src/stores/useRobotStore.ts</files>

1. Create `src/types/robot.ts` with:
   - `RobotEmotion` type: union of `'idle' | 'happy' | 'sad' | 'excited' | 'thinking'`
   - `EMOTION_CLIP_MAP`: `Record<RobotEmotion, string>` mapping each emotion to the animation clip name in the .glb file. For the placeholder dragon (which has only one clip named "Take 001"), map ALL emotions to "Take 001". Include a comment: "Update these clip names when the actual robot model is provided. Run `console.log(gltf.animations.map(a => a.name))` to discover clip names."
   - `RobotState` interface: `{ emotion: RobotEmotion; isModelLoaded: boolean; performanceTier: 'high' | 'low'; setEmotion: (emotion: RobotEmotion) => void; setModelLoaded: (loaded: boolean) => void; setPerformanceTier: (tier: 'high' | 'low') => void; }`
   - `MODEL_PATH` constant: `'/models/robot.glb'`

2. Create directory `src/stores/` and file `src/stores/useRobotStore.ts` with:
   - Import `create` from `zustand`
   - Import `RobotEmotion` and `RobotState` from `@/types/robot`
   - Create and export `useRobotStore` using `create<RobotState>`:
     - `emotion: 'idle'` (default)
     - `isModelLoaded: false`
     - `performanceTier: 'high'`
     - `setEmotion: (emotion) => set({ emotion })`
     - `setModelLoaded: (loaded) => set({ isModelLoaded: loaded })`
     - `setPerformanceTier: (tier) => set({ performanceTier: tier })`

Both files must use `'use client'` is NOT needed -- these are pure TypeScript modules with no React hooks at the top level. Zustand `create` is a plain function call, not a hook. The store becomes a hook only when consumed.

3. Run `npm run build` to confirm these new files do not break the build. They contain no Three.js imports, so there is no SSR risk.
