# Fix Project Section Overflow — Add Container Wrapper

**Type:** Quick Fix
**Created:** 2026-04-09

## Problem

The `ProjectsSection` component already has internal `overflow-hidden` handling via its `<section>` tag padding, but the section itself is missing the `overflow-hidden` wrapper that other sections (like `AboutSection`) use. The `ProjectCardParallax` animations and the masonry grid column offsets (`md:pt-[80px]`, `lg:pt-[160px]`) can cause horizontal or vertical overflow that bleeds outside the section boundaries.

Additionally, in `page.tsx`, the `ProjectsSection` is rendered directly without a wrapping `<div>` container, unlike `AboutSection` and `AchievementsSection` which are wrapped in `<div id="...">` containers.

## Analysis

- **`AboutSection`** — has `className="overflow-hidden"` on its `<section>` tag
- **`AchievementsSection`** — wrapped in `<div id="achievements">` in page.tsx
- **`ProjectsSection`** — rendered bare: `<ProjectsSection projects={...} locale={lang} />`
  - No `overflow-hidden` on its `<section>` tag
  - No wrapping `<div>` in page.tsx

## Tasks

### Task 1: Add `overflow-hidden` to ProjectsSection and wrap in page.tsx

**File:** `src/components/projects/projects-section.tsx`
- Add `overflow-hidden` to the `<section>` className to prevent parallax/grid overflow

**File:** `src/app/[lang]/page.tsx`
- Wrap `<ProjectsSection>` in a `<div>` container for consistency with other sections (no id needed since the section already has `id="projects"`)

- Run `npm run build` or `npx next lint` to verify no errors
